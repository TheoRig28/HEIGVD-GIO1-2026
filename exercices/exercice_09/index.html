Exercice 09

Cet exercice vous guide dans l'installation de Node.js et la création de votre première application avec OpenLayers


  1. Si vous n'avez pas Node.js sur votre machine, téléchargez-le ici et installer l'application en laissant toutes les options par défaut
  2. Une fois Node.js installé, ouvrez Visual Studio Code et ouvrez le dossier de cet exercice (menu Fichier, Ouvrir un dossier)
  3. Ouvrez un terminal dans le dossier de cet exercice. Cela peut être fait depuis Visual Studio Code avec le menu "Terminal", puis "Nouveau terminal"
  4. Dans le terminal, tapez les commandes indiquées ici
    • npm create ol-app my-app indique à Node.js de créer une application de base OpenLayers. Node.js créera un nouveau dossier my-app contenant le code minimal pour faire fonctionner OpenLayers
    • cd my-app indique de déplacer le terminal dans le dossier my-app
    • npm start indique de démarrer l'application avec Vite. Vite va créer un serveur de développement exécutant l'application
    • Vite affichera dans la console une ligne semblable à Local: http://localhost:5173/, il s'agit de l'adresse à taper dans votre navigateur pour lancer l'application
  5. En vous rendant à l'adresse indiquée par Vite, vous devriez voir une carte OpenStreetMap en plein écran
  6. Le serveur de développement de Vite est très pratique. Vite actualisera automatique votre page web si vous modifiez un des fichiers de l'application (fichier HTML ou JavaScript)
  7. Consultez les fichiers / dossiers créés dans votre projet
    • package.json contient la description de votre projet Node.js. C'est là que peuvent être indiqués le nom du projet, son auteur, sa version, etc. Ce fichier contient aussi la liste des dépendances que vous avez explicitement installées. Vous pouvez éditer ce fichier manuellement. Dans sa partie "scripts", ce fichier indique les commandes du projet. Pour cet exemple, il existe les commandes
      • npm run start (ou son équivalent raccourci npm start) qui démarre Vite en mode test, utile pour développer en local sur votre machine
      • npm run build qui produira le code final, prêt à être mis en production, dans le dossier dist
      • npm run serve qui démarre un serveur local qui sert le contenu du dossier dist, pour tester le résultat de npm run build
    • package-lock.json contient la liste des dépendances que vous n'avez pas explicitement installé, mais qui sont requises pas d'autres librairies. Ce fichier n'est pas à éditer manuellement, il est (re)créé lors de l'installation du projet avec npm install
    • vite.config.js contient les paramètres relatifs à Vite
    • le dossier node_modules qui contient toutes les dépendances du projet. Il ne faut pas l'inclure avec git (notez qu'il est exclus dans le .gitignore par défaut) car ce dossier est recréé à chaque nouveau npm install

Une fois que vous avez créé une application de base fonctionnelle et compris comment Node.js organise ses projets, vous pouvez passer à l'exercice suivant.
