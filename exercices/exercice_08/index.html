Exercice 08

Un catalogue STAC est une manière d'organiser et de diffuser des métadonnées relatives à des données géographiques.

De manière simplifiée, un catalogue STAC est composé d'un ensemble de fichiers JSON, organisés en hiérarchie, qui permettent de naviguer dans un ensemble de métadonnées.

Voici quelques exemples de catalogues STAC :

  • Swisstopo : https://data.geo.admin.ch/browser/index.html
  • Canton de Lucerne : https://daten.geo.lu.ch/browser#/
  • Canton de Neuchâtel : https://sitn.ne.ch/web/stac
Ces interface sont simplement une visualisation des fichiers JSON qui composent le catalogue. Il est possible de consulter directement les fichiers JSON :
  • Pour Swisstopo : https://data.geo.admin.ch/api/stac/v0.9/
  • Pour Lucerne : https://daten.geo.lu.ch/api/stac/v1.0/
  • Pour Neuchâtel : https://sitn.ne.ch/services/stac/catalog.json


Les catalogues STAC sont composés (entre autres) de deux types d'objets :
  1. Les collections, qui décrivent un ensemble d'éléments géographiques.
    Une collection ressemble à ceci (certains propriétés sont omises pour simplifier) : 
    {
    "type": "Catalog",
    "title": "Titre de la collection"
    "description": "Contenu de la description",
    "links": [
        {
            "rel": "root",
            "href": "https://sitn.ne.ch/services/stac/catalog.json",
            "type": "application/json",
            "title": "Catalogue STAC du SITN"
        },
        {
            "rel": "child",
            "href": "https://url/vers/une/collection/enfant.json",
            "type": "application/json",
            "title": "Titre de la sous-collection"
        },
        {
            "rel": "item",
            "href": "https://url/vers/un/item.json",
            "type": "application/geo+json"
        },

        {
            "rel": "collections",
            "href": "https://url/vers/toutes/les/collections/du/catalogue.json",
            "type": "application/json"
        },
        {
            "rel": "self",
            "href": "https://url/vers/la/page/actuelle.json",
            "type": "application/json"
        }
    ],
}
    Une collection contient un titre, une description, et un ensemble de liens parmi lesquels les liens de type "child" sont les collections enfant. Les liens de type "item" sont les liens vers les items de la collection. Il peut y avoir plusieurs collections enfants et plusieurs items par collection. Il existe d'autres types de liens que des "child" et "item", comme "root" qui indique la racine du catalogue.
  2. Les items, qui décrivent un élément géographique.
    Un item ressemble à ceci (certaines propriétés sont omises pour simplifier): 
    {
  "type": "Feature",
  "id": "identifiant_de_item",
  "geometry": { /* type de géométrie, ne nous intéresse pas pour cet exercice */ },
  "links": [ /* ensemble de liens, ne nous intéresse pas pour cet exercice */ ],
  "assets": { /* fichiers composants l'item. ne nous intéresse pas pour cet exercice */ },
}

L'objectif de cet exercice est d'écrire une interface web capable de récupérer et d'afficher certaines parties d'un catalogue STAC.
L'interface ci-dessous permet d'indiquer l'URL d'une collection. Au clic sur le bouton, l'application doit
  1. Charger la collection indiquée, puis afficher son titre et sa description dans la section "Titre et description"
  2. Lister dans la section "Sous-Collections" toutes les collection enfant de la collection indiquée
  3. Pour chaque item indiqué dans les liens de la collection, l'application récupère l'item et affiche son id dans une liste
  4. Lors du chargement des items, l'application modifie le titre de la section pour indiquer la progression du chargement, par exemple "Items (10/35)"
  5. Le bouton reset efface le contenu des sections
  6. Le chargement des items doit se faire de manière asynchrone

Voici quelques exemples pour tester :
  • Collection "Lidar 2022" de Neuchâtel, contient 11 sous-collections: https://sitn.ne.ch/services/stac/ne/lidar2022/collection.json
  • Modèles de surface 2022 de Neuchâtel, contient 3 items : https://sitn.ne.ch/services/stac/ne/lidar2022/lidar2022mns/collection.json
  • Lise des tuiles LiDAR 2022 de Neuchâtel, contient 3218 items : https://sitn.ne.ch/services/stac/ne/lidar2022/pointcloud/collection.json
