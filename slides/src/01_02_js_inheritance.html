L'héritage

Supposons avoir une classe Date


        class Date {
            constructor(year, month, day){
                this.year = year;
                this.month = month;
                this.day = day;
            }
            toText(){
                return `${this.day}.${this.month}.${this.year}`;
            }
            print(){
                console.log(`Valeur actuelle : ${this.toText()}`)
            }
        }

On souhaite maintenant créer une deuxième classe DateTime, stockant en plus l'heure. Il serait bien de pouvoir réutiliser la classe Date, sans devoir réimplémenter tout son contenu.

JavaScript

L'héritage

JavaScript permet de le faire de la manière suivante


        class DateTime extends Date {    // Le mot clé "extends" indique l'héritage
            constructor(year, month, day, hours, minutes, seconds){
                super(year, month, day) // "super" est le constructeur de la classe parente
                this.hours = hours;
                this.minutes = minutes;
                this.seconds = seconds;
            }
            toText(){
                // "super" permet également d'appeler des fonctions de la classe parente
                return super.toText() + ` ${this.hours}:${this.minutes}:${this.seconds}`
            }
        }

L'héritage

On aura donc le comportement suivant


        const date = new Date(2010, 11, 12);
        console.log(date.toText()) // 12.11.2010
        date.print(); // affiche : Valeur actuelle : 12.11.2010

        const dateTime = new DateTime(2020, 10, 25, 22, 35, 45);
        console.log(dateTime.toText()) // 25.10.2020 22:35:45
        dateTime.print(); // affiche : Valeur actuelle : 25.10.2020 22:35:45

A noter :

  • Les méthodes et attributs de la classe parente sont disponibles dans la classe enfant.
    Un DateTime peut appeler print() définit dans la classe Date.
  • La classe enfant peut "écraser" les fonctions de la classe parente.
    DateTime redéfinit la fonction toText, pour y inclure l'heure
    • Dans ce cas print() appellera bien toText() de la classe enfant, même si print() n'est jamais définit dans la classe DateTime
